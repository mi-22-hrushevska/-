# 💻 Поглиблений практикум: Адміністрування та інженерія ОС

Цей блок містить 3 лабораторні роботи та один практичний інженерний кейс, які дозволять вам перейти від теорії до реального керування ресурсами комп'ютера.

## 🧪 Лабораторна робота №1: Глибокий моніторинг ресурсів та пошук "Вузьких місць" (Bottlenecks)

**Мета:** Навчитися діагностувати апаратні проблеми ОС, аналізувати витоки пам'яті (Memory Leaks) та оцінювати роботу планувальника CPU.

### 🛠️ Хід роботи:
1.  **Аналіз черги процесора:** 
    *   У Windows натисніть `Win + R`, введіть `resmon` (Монітор ресурсів) і перейдіть на вкладку **ЦП (CPU)**. (В Linux запустіть `htop`).
    *   Знайдіть стовпчик **PID (Process Identifier)**. Запишіть у звіт, що це таке і чому кожен PID є унікальним.
    *   Зверніть увагу на параметр **Потоки (Threads)**. Знайдіть процес із найбільшою кількістю потоків. Чому саме ця програма потребує так багато потоків?
2.  **Діагностика фрагментації та віртуальної пам'яті:**
    *   Перейдіть на вкладку **Пам'ять**. Зверніть увагу на колірну діаграму.
    *   Знайдіть параметр "Виділено" (Commit Charge). Яку частину займає фізична RAM, а яку — файл підкачки?
3.  **Експеримент "Стрес-тест":**
    *   Відкрийте в браузері 15-20 важких закладок одночасно (наприклад, з відео високої якості).
    *   Зафіксуйте в Моніторі ресурсів, як ОС починає переміщувати сторінки пам'яті з RAM на диск. Опишіть, як це вплинуло на швидкість відгуку системи.

## 🐚 Лабораторна робота №2: CLI як інструмент автоматизації (Кросплатформений рівень)

**Мета:** Освоїти інтерфейс командного рядка для керування процесами та файловими системами без графічного інтерфейсу.

### 📊 Шпаргалка системних команд (Зіставлення ОС):

| Дія | Команда Windows (CMD / PowerShell) | Команда Linux / macOS (Bash) |
| :--- | :--- | :--- |
| Перегляд інформації про ОС | `systeminfo` | `uname -a` або `lscpu` |
| Список усіх процесів у системі | `tasklist` | `ps aux` |
| Примусове завершення процесу | `taskkill /PID [номер] /F` | `kill -9 [номер_PID]` |
| Перевірка цілісності диска та FS | `chkdsk C: /f` | `df -h` та `fsck` |

### 🛠️ Хід роботи:
1.  Відкрийте термінал вашої ОС.
2.  Виведіть на екран повну технічну інформацію про вашу систему за допомогою команди зі таблиці. Скопіюйте у звіт версію ядра вашої ОС.
3.  **Створення скрипту автоматизації:**
    *   *Якщо ви на Windows:* Створіть у папці `practice/` файл `cleanup.bat` і запишіть туди:
        ```bat
        echo Очищення тимчасових файлів ОС...
        del /q /f /s %temp%\*
        echo Системні процеси після очищення:
        tasklist
        pause
