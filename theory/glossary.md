# 📖 Розширений глосарій термінів з архітектури ОС

Цей глосарій містить ключові поняття та підсистеми сучасних операційних систем, які необхідні для глибокого розуміння матеріалу модуля.

| Термін | Визначення |
| :--- | :--- |
| **Ядро ОС (Kernel)** | Центральна, найважливіша частина операційної системи, яка працює в захищеному режимі та безпосередньо керує процесором, пам'яттю та залізом. |
| **Драйвер** | Спеціалізований програмний модуль, який виступає "перекладачем" між операційною системою та конкретним апаратним пристроєм. |
| **Процес (Process)** | Програма, яка виконується в даний момент часу, для якої ОС виділила ізольований простір в оперативній пам'яті та системні ресурси. |
| **Потік (Thread)** | Найменша одиниця виконання всередині процесу. Один процес може породити десятки потоків, які ділять між собою його пам'ять (наприклад, вкладки в браузері). |
| **Переривання (Interrupt)** | Сигнал від апаратного чи програмного забезпечення, який змушує процесор негайно призупинити поточний код і перейти до виконання екстреного завдання ОС. |
| **Системний виклик (System Call)** | Інтерфейс (запит), за допомогою якого прикладна програма в режимі користувача просить ядро ОС виконати привілейовану дію (наприклад, записати файл чи увімкнути камеру). |
| **Планувальник завдань (Scheduler)** | Компонент ядра ОС, який за певним алгоритмом визначає черговість та тривалість виконання процесів на кожному ядрі процесора. |
| **Round Robin (Карусель)** | Алгоритм планування, за якого кожному процесу по черзі виділяється фіксований маленький квант часу процесора (типово 10–20 мс). |
| **Взаємне блокування (Deadlock)** | Ситуація в багатозадачних системах, коли кілька процесів повністю зупиняються (зависають), бо кожен із них безкінечно чекає на ресурс, захоплений іншим процесом. |
| **Фрагментація пам'яті** | Проблема, за якої в оперативній пам'яті утворюється багато дрібних вільних ділянок, через що неможливо виділити безперервний великий блок для нової програми. |
| **Сторінкова організація (Paging)** | Метод керування пам'яттю, за якого ОС розбиває віртуальну пам'ять програми та фізичну RAM на однакові блоки (сторінки), позбавляючись проблеми фрагментації. |
| **Віртуальна пам'ять** | Технологія, яка створює для кожної програми ілюзію наявності великої безперервної оперативної пам'яті, навіть якщо фізично вона розкидана шматками по RAM та диску. |
| **Файл підкачки (Swap / Paging file)** | Спеціальний файл на жорсткому диску або SSD, куди ОС скидає тимчасово невикористовувані сторінки пам'яті з RAM, коли фізичної оперативки не вистачає. |
| **Файлова система (FS)** | Набір правил та структур даних, які визначають, як саме інформація групується, іменується та зберігається на фізичному носії (диску). |
| **Журналювання файлової системи** | Технологія, за якої зміни спочатку записуються в спеціальний журнал, а вже потім вносяться в структуру диска. Захищає дані від пошкодження під час раптового вимкнення живлення. |
| **Режим ядра (Kernel Mode)** | Привілейований режим роботи процесора, у якому код (ядро ОС) має безмежний доступ до всієї пам'яті та будь-яких команд заліза. |
| **Режим користувача (User Mode)** | Обмежений режим роботи процесора для звичайних програм. Захищає операційну систему від збоїв або шкідливих дій прикладного софту. |
